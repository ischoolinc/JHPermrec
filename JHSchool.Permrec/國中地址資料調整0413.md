# 國中地址資料調整 0413

## 1. 問題現象

- 首次載入學生地址資料到畫面時，`txtZipcode` 會由原本資料庫值，被 `cboCounty / cboTown` 初始化事件回填邏輯覆寫成對照表的郵遞區號。

## 2. 原因分析

- `BindDataToForm()` 綁值 `cboTown.Text` 時會觸發 `cboTown_TextChanged()`。
- `cboTown_TextChanged()` 會呼叫 `CheckTownChange()`，而 `CheckTownChange()` 依縣市/鄉鎮市區對照表反向改寫 `txtZipcode.Text`。

## 3. 修改方式

- 在 `AddressPalmerwormItem.BindDataToForm()` 進行畫面欄位綁值期間，加入既有保護旗標 `_date_updating = true`。
- 使用 `try/finally` 確保綁值結束或中途例外時，皆會恢復 `_date_updating = false`。

## 4. 影響範圍

- 只影響 **初始化綁值期間** 的事件誤觸發。
- 不影響使用者手動變更縣市、鄉鎮市區時自動同步郵遞區號的原本行為。
- 不影響使用者手動輸入郵遞區號後反查縣市、鄉鎮市區的原本行為。

## 5. 測試結果

- 首次載入可保留原始 `ZipCode`，不會被 town mapping 覆寫。
- 手動修改 `cboTown` 仍會依 mapping 同步 `txtZipcode`。
- 手動輸入 `txtZipcode`（Enter 或離開欄位）仍可反查回填 `cboCounty / cboTown`。
- 儲存功能正常，`OnSaveButtonClick()` 寫回流程不受影響。

## 二、取消按鈕（0413-2）

### 問題

取消按鈕只隱藏儲存／取消按鈕，不會將畫面還原成 `_StudAddressRec` 目前資料，造成「假取消」：畫面仍顯示未儲存修改。

### 原因

`OnCancelButtonClick` 僅設定 `SaveButtonVisible` / `CancelButtonVisible`，未重新載入畫面。

### 修正

在 `OnCancelButtonClick` 先呼叫 `BindDataToForm()`，再隱藏按鈕：

```csharp
BindDataToForm();
SaveButtonVisible = false;
CancelButtonVisible = false;
```

`BindDataToForm()` 會依 `_StudAddressRec` 重綁畫面、Reset ChangeListener，且綁值期間已有 `_date_updating` 保護，不影響既有 ZipCode／縣市鄉鎮邏輯。

### 結果

- 取消可正確還原欄位。
- 不影響儲存、ZipCode 反查、Town 同步等既有行為。
- UI 按鈕狀態與資料一致。
